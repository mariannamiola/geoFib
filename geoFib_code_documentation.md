# geoFib Code Documentation and User Manual

**Repository:** `geoFib`  
**Manuscript:** *A Fibonacci-Like Propagation Graph for Infiltration and Internal Erosion in Heterogeneous Layered Soils*  
**Code purpose:** reproducible demonstration of the interface-aware reservoir Fibonacci reduced-order model (ROM) on a controlled Gaussian storm scenario.

---

## 1. Overview

The repository contains a compact Python implementation of the Fibonacci geo-hydraulic ROM described in the manuscript. The code constructs a directed acyclic Fibonacci graph, assigns synthetic layered soil properties, runs the ROM under a Gaussian surface forcing, and exports diagnostic metrics, CSV files, and figures.

The distributed demo is self-contained. It does not require HYDRUS-1D, external solvers, external datasets, or compiled extensions. The HYDRUS-1D benchmark discussed in the manuscript is not part of this minimal demonstration repository.

## 3. Software requirements

Python 3.10 or newer is recommended.

Required third-party packages:

```text
numpy>=1.23
matplotlib>=3.6
```

No compilation is required. The code is pure Python and can be executed directly from the command line.

## 4. Installation

From a terminal, move into the repository directory and install the required packages.

```bash
cd geoFib-main
python -m pip install -r requirements.txt
```

A virtual environment is recommended but not required. For example:

```bash
python -m venv .venv
source .venv/bin/activate      # Linux/macOS
python -m pip install -r requirements.txt
```

On Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## 5. Running the demo

The standard execution command is:

```bash
python run_synthetic_demo.py --out outputs_demo
```

This command runs a seven-level Fibonacci graph under a Gaussian storm and writes all results to the directory `outputs_demo`.

A fully explicit command equivalent to the default configuration is:

```bash
python run_synthetic_demo.py --out outputs_demo --levels 7 --steps 49 --seed 7
```

---

## 6. Command-line options

The script `run_synthetic_demo.py` accepts the following options.

| Option | Type | Default | Meaning |
|---|---:|---:|---|
| `--out` | string | `outputs_demo` | Output directory where CSV files, figures, and `metrics.json` are written. |
| `--levels` | integer | `7` | Number of Fibonacci graph levels. With `--levels 7`, the graph has 33 nodes. |
| `--steps` | integer | `49` | Number of hourly output steps. The default corresponds to 49 sampled time points. |
| `--seed` | integer | `7` | Random seed used to generate reproducible within-level variability in synthetic soil properties. |
| `-h`, `--help` | flag | - | Show the command-line help message. |

To show the available options directly:

```bash
python run_synthetic_demo.py --help
```

### CSV and JSON outputs

| Output file | Description |
|---|---|
| `metrics.json` | Summary scalar diagnostics: number of levels, number of nodes, peak wetting-front level, peak activated nodes, time of peak activation, maximum and mean exceedance ratio, and deepest-level attenuation. |
| `forcing.csv` | Time series of the Gaussian root forcing. Columns: `time_h`, `forcing`. |
| `wetting_front.csv` | Time series of the cumulative wetting-front level. Columns: `time_h`, `wetting_front_level`. |
| `activated_nodes.csv` | Time series of the number of nodes whose pressure exceeds the local threshold. Columns: `time_h`, `activated_nodes`. |
| `level_mean_theta.csv` | Level-wise mean wetting/reservoir state over time. Columns: `time_h`, `level_0`, ..., `level_L-1`. |
| `level_mean_pressure.csv` | Level-wise mean pressure-like response over time. Columns: `time_h`, `level_0`, ..., `level_L-1`. |
| `attenuation_profile.csv` | Level-wise normalised peak pressure attenuation. Columns: `level`, `normalised_peak_pressure`. |
| `exceedance_ratio_by_node.csv` | Node-level pressure-to-threshold exceedance ratio. Columns: `node`, `level`, `C`. |

## 8. Model workflow implemented by the demo

The demo performs the following steps:

1. Build a directed Fibonacci graph with `L` levels and level populations `|V_k| = F_{k+1}`.
2. Assign reproducible synthetic geotechnical properties to each node: porosity, saturated hydraulic conductivity, and pressure threshold.
3. Generate a Gaussian surface forcing at the root node.
4. Propagate the transient pressure-like signal and erosion-propensity proxy through the Fibonacci recurrence.
5. Update the wetting component using a local reservoir law with an interface-aware hydraulic-contrast correction.
6. Compute diagnostic quantities: wetting-front depth, activated-node count, pressure-to-threshold exceedance ratio, level-wise mean states, and attenuation profile.
7. Export CSV files, summary metrics, and figures.

---

## 9. Main diagnostic quantities

The code reports the following model diagnostics.

| Diagnostic | Code output | Meaning |
|---|---|---|
| Wetting-front level | `wetting_front.csv` | Deepest graph level where the wetting/reservoir state exceeds a relative threshold. |
| Activated nodes | `activated_nodes.csv` | Number of nodes with pressure above the local pressure threshold at each time. |
| Exceedance ratio `C(v)` | `exceedance_ratio_by_node.csv` | Node-wise maximum pressure divided by the local pressure threshold. Values above 1 indicate at least one threshold exceedance. |
| Attenuation profile | `attenuation_profile.csv` | Level-wise peak pressure normalised by the root-level peak pressure. |
| Level means | `level_mean_theta.csv`, `level_mean_pressure.csv` | Mean wetting and pressure signals aggregated by graph depth level. |

The erosion-propensity component is a synthetic proxy. It should not be interpreted as calibrated cumulative erosion damage.

## 10. Reproducibility

The demo is deterministic for a fixed set of command-line options. In particular, the synthetic node properties are controlled by the `--seed` option.

Example reproducible run:

```bash
python run_synthetic_demo.py --out outputs_demo_seed7 --levels 7 --steps 49 --seed 7
```

Changing the seed changes the within-level heterogeneity of the synthetic soil properties while preserving the same graph construction and model equations.

## 11. Modifying the experiment

The simplest way to modify the demonstration is through command-line options:

```bash
python run_synthetic_demo.py --out outputs_L8 --levels 8 --steps 72 --seed 11
```

For deeper modifications, edit the following functions in `fibonacci_rom_core.py`:

| Function or class | Purpose |
|---|---|
| `ROMParameters` | Stores recurrence, reservoir, interface, and wetting-front threshold parameters. |
| `build_fibonacci_graph()` | Constructs the Fibonacci graph and parent maps. |
| `make_synthetic_layered_properties()` | Defines the synthetic layered soil profile and node-level variability. |
| `gaussian_forcing()` | Defines the Gaussian storm forcing. |
| `run_rom()` | Runs the ROM recurrence and reservoir update. |
| `wetting_front_depth()` | Computes the wetting-front diagnostic. |
| `activated_nodes()` | Computes activated-node counts. |
| `exceedance_ratio()` | Computes the node-wise pressure-to-threshold exceedance ratio. |
| `attenuation_profile()` | Computes the depth-level attenuation profile. |

## 12. Notes on interpretation

This repository is intended as a transparent computational companion to the manuscript. It demonstrates the ROM structure and its diagnostic outputs on a controlled synthetic case.

The parameters are not calibrated to a specific field site. The pressure variable is a dimensionless pressure-like signal, and the exceedance ratio is a threshold indicator. The wetting component uses a bounded reservoir state for wetting-front screening. The code should therefore be used for reproducibility, inspection, and extension of the proposed reduced-order modelling framework, rather than as a site-specific geotechnical design tool.

## 13. Troubleshooting

### `ModuleNotFoundError: No module named 'numpy'` or `matplotlib`

Install the dependencies:

```bash
python -m pip install -r requirements.txt
```

### Output directory already exists

The script can write into an existing directory. Existing files with the same names may be overwritten. To preserve previous outputs, use a new output directory:

```bash
python run_synthetic_demo.py --out outputs_new_run
```

### Figures are not displayed on screen

The script saves figures to PNG files and closes them. This is expected behaviour for batch execution and reproducible output generation.

### Different results after changing `--seed`

This is expected. The seed controls the random within-level variability of the synthetic soil properties.
